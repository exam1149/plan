How I Plan To Gather Feedback
Greenfield Local Hub

Aim
The aim of this plan is to collect feedback on the Greenfield Local Hub website in order to evaluate its usability, design, functionality, and overall user experience. The feedback will be used to identify areas for improvement and ensure the website meets user needs.
Target Audience
Feedback will be collected from both technical and non-technical users. This is to ensure a balanced evaluation from different perspectives. 
Methods of Data Collection
Google Forms
Two separate google forms will be created, for technical and non-technical users. These forms will include a mixture of questions and the responses will be collected and analysed.
Data Analysis
The collected data will be analysed by: 
-	Reviewing multiple choice responses
-	Grouping similar feedback together
-	Identifying common issues reported by multiple users
Feedback will be grouped into:
-	Usability issues
-	Design issues
-	Functional errors
-	Performance concerns
Use of Feedback
The feedback will be used to improve the website by:
-	Fixing bugs
-	Improving the layout and design
-	Making navigation clearer
-	Enhancing overall user experience
Limitations
A few limitations to consider when collecting data:
-	A limited number of participants may reduce the reliability of results
-	Technical feedback may be limited depending on available participants
Success Criteria
The feedback collection can be considered successful if:
-	A sufficient (5-10) number of responses are collected
-	Both technical and non-technical users provide feedback
-	Clear areas for improvement are identified
Conclusion
This plan ensures that feedback is collected using multiple methods and from different types of users. The results will provide valuable insights that can be used to improve the functionality, usability, and design of the Greenfield Local Hub website.
